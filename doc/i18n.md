# Internationalization (i18n)

The Aropixel Blog Bundle natively supports multi-language content. This feature is automatically enabled based on your project's configuration.

### 1. Enabling i18n

Internationalization is enabled when more than one locale is configured in the `aropixel_admin` configuration.

In your `config/packages/aropixel_admin.yaml`:

```yaml
aropixel_admin:
    # i18n is enabled if there is more than one locale here
    locales: ['fr', 'en']
```

If only one locale is defined, the blog will operate in single-language mode using the standard `PostType` form. If two or more locales are defined, it will automatically switch to the multilingual mode using the `PostTranslatableType` form.

### 2. Customizing the Translatable Form

If you need to customize the form used for translatable posts (for example, to add fields or change their order), you can specify your own form class in the `aropixel_blog.yaml` configuration using the `post_translatable` parameter.

In `config/packages/aropixel_blog.yaml`:

```yaml
aropixel_blog:
    forms:
        post_translatable: App\Form\MyCustomPostTranslatableType
```

By default, it uses `Aropixel\BlogBundle\Form\PostTranslatableType`.

### 3. Database Requirements

When i18n is enabled, ensure that your database schema is up to date, as it uses the `aropixel_post_translation` and `aropixel_post_category_translation` tables to store translated content.

```bash
php bin/console doctrine:schema:update --force
```
